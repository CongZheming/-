# 跨平台舆情材料识别与归类系统

本项目是一个本地运行的 Streamlit 科研辅助工具，服务于“社交媒体中的网络舆情发展过程和特点研究：以生育政策相关议题的跨平台传播与互动为例”。系统采用“人工收集材料 + 系统识别归类 + 人工复核确认”的流程，不做自动爬虫，不绕过平台限制。

## 功能说明

- 文本粘贴录入微博、小红书、知乎、豆瓣、抖音等平台材料。
- 预留截图上传和 PaddleOCR 中文识别接口；OCR 不可用时可手动补全文本。
- 自动清洗文本，去除 URL、重复空行和常见界面噪声。
- 使用关键词词典和简单得分完成议题相关性、内容类型、情绪、话语框架、平台角色、议题重塑类型推荐。
- 支持人工复核并保存最终标签。
- 使用 SQLite 本地保存材料、分类、OCR 结果和标签字典。
- 支持数据浏览、筛选、基础统计和 Excel / CSV 导出。
- 支持删除误录材料或单条分类记录；删除材料时会同步删除关联 OCR 和分类记录。

## 环境安装

建议使用 Python 3.10 或更高版本。

```bash
cd opinion_material_classifier
pip install -r requirements.txt
```

如需截图 OCR，可额外安装 PaddleOCR。PaddleOCR 依赖较多，安装失败不影响 V1.0 的文本粘贴输入和规则分类功能。

```bash
pip install paddleocr
```

## 启动方式

```bash
streamlit run app.py
```

也可以在 Windows 文件夹中双击 `run_app.bat` 启动。首次运行时如果缺少 Streamlit，脚本会尝试执行 `pip install -r requirements.txt`。

启动后在浏览器中打开 Streamlit 提供的本地地址。

## 使用流程

1. 在“材料上传 / 文本录入”页面选择平台、材料类型、输入方式并录入文本。
2. 点击“保存并进行分类”，系统保存材料。
3. 在“自动识别与归类”页面查看最近一条未分类材料，保存系统推荐分类。
4. 在“人工复核”页面查看推荐标签，手动修正并确认最终标签。
5. 在“数据浏览”页面查看和筛选材料与分类数据。
6. 在“数据删除”页面删除误录材料或错误分类记录。建议删除前先导出备份。
7. 在“数据导出”页面导出 Excel 和 CSV 文件。
8. 在“统计概览”页面查看平台、材料类型、情绪、话语框架和重塑类型分布。

## 数据库说明

SQLite 数据库位于：

```text
data/opinion_materials.db
```

主要数据表：

- `materials`：保存原始材料、清洗文本、截图路径和研究者备注。
- `classifications`：保存系统推荐分类和人工复核后的最终标签。
- `ocr_results`：保存截图 OCR 的原始识别结果和修正文本。
- `label_dictionary`：保存内置标签体系。

## 标签体系说明

- 议题相关性：强相关、中等相关、弱相关、无关。
- 内容类型：政策讨论、专家质疑、个人经验、情绪宣泄、反讽调侃、理性分析、平台审核规避、二次创作、媒体传播、其他。
- 情绪标签：支持、反对、焦虑、愤怒、无奈、嘲讽、调侃、恐惧、理性分析、中立、复杂 / 混合。
- 话语框架：生育成本、女性身体风险、性别不平等、专家权威质疑、政策信任、政策质疑、婚恋压力、家庭责任、职场歧视、育儿资源、人口焦虑、代际观念、媒体标题党、平台审核规避、科学研究方法质疑、个人经验叙事、其他。
- 平台角色：话题定义、热度扩散、情绪发酵、经验分享、理性沉淀、二次创作、梗文化生成、政策解释、反讽抵抗、审核规避、其他。
- 议题重塑类型：标题化、情绪化、经验化、性别化、娱乐化、专业化、政策化、道德化、反讽化、审核规避化、其他。
